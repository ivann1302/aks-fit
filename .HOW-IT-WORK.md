# Как это работает

Объяснения концепций, используемых в проекте — для джунов.

---

## FSD (Feature-Sliced Design)

Архитектура проекта. Код делится на слои по степени абстракции:

```
src/
  app/        — точка входа, глобальные настройки (layout, стили)
  pages/      — страницы (собирают виджеты)
  widgets/    — большие самостоятельные блоки UI (Header, Footer, калькулятор)
  features/   — действия пользователя (пока пусто)
  entities/   — бизнес-сущности (пока пусто)
  shared/     — общее: UI-кит, хуки, утилиты
```

**Правило:** нижний слой не знает о верхнем. `shared` не импортирует из `widgets`.

**Изолированная точка импорта** — каждый модуль экспортирует через `index.ts`:

```ts
// widgets/calculator/index.ts
export { CalculatorSection } from './ui/CalculatorSection';
```

Снаружи пишем `import { CalculatorSection } from '@/widgets/calculator'`, а не лезем внутрь папки.

## Next.js (App Router)

`src/app/` — это App Router Next.js 13+.

- `layout.tsx` — общая обёртка для всех страниц (шрифты, мета, глобальные стили)
- `page.tsx` — рендерит конкретную страницу
- `'use client'` — директива, которая говорит Next.js: «этот компонент работает в браузере» (нужна для `useState`, `useEffect`, событий)

По умолчанию все компоненты в App Router — **Server Components** (рендерятся на сервере). Как только нужен стейт или браузерное API — добавляй `'use client'`.

## SCSS Modules

Файлы вида `Component.module.scss`. Каждый класс получает уникальное имя при сборке, поэтому классы не конфликтуют между компонентами.

```tsx
import styles from './Button.module.scss';
// styles.button → что-то вроде "Button_button__xK2p"
<button className={styles.button}>
```

**`@use` вместо `@import`**
Современный способ подключить переменные/миксины из другого файла:

```scss
@use '../../../styles/variables' as *; // as * = без namespace, пишем $color-orange, не variables.$color-orange
```

## Parallax-скролл (`useParallaxScroll`)

Хук реализует эффект параллакса вручную:

1. `container` — блок с фиксированной высотой (равной всему контенту). Браузер скроллит его.
2. `main` — `position: fixed`, двигается вверх через `top: -scrollY` в обработчике скролла.
3. `ResizeObserver` — следит за изменением высоты контента (например, когда появляется результат калькулятора) и пересчитывает высоту контейнера.

```
пользователь скроллит → window.scrollY растёт → main.style.top уменьшается → контент «едет» вверх
```

## Формула Миффлина–Сан Жеора (калькулятор калорий)

Рассчитывает базовый обмен веществ (BMR — сколько калорий тратится в покое):

- Мужчины: `10 × вес + 6.25 × рост − 5 × возраст + 5`
- Женщины: `10 × вес + 6.25 × рост − 5 × возраст − 161`

BMR умножается на КФА (коэффициент физической активности) → получаем TDEE (суточная норма с учётом активности).

Затем корректируется по цели: −500 ккал для похудения, +500 для набора массы.
